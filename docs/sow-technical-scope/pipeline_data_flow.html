Document Extraction Pipeline - Data Flow
End-to-end flow from carrier document ingestion to validated structured output
1
Document Ingestion
Cloud Run + Cloud Storage + Pub/Sub
  • Document received via API upload, email webhook, or scheduled portal pull
  • Stored in Cloud Storage with carrier-partitioned bucket structure
  • Processing event published to Pub/Sub queue
  • Metadata record created in Cloud SQL (carrier_id, doc_type_hint, source, timestamp)
▼
2
Document Classification
Vertex AI (Gemini) - Multi-Modal
  • Gemini analyzes document visually + text content to classify type
  • Categories: Invoice, CSR Record, Contract, Portal Export, Operational Report
  • Classification confidence score assigned (0.0 - 1.0)
  • If confidence < 0.8: flagged for human review of classification
  • Carrier identification: match to known carrier profile in config store
▼
3
Carrier Config Lookup
Cloud SQL (PostgreSQL)
  • Load carrier-specific extraction template (field list, expected positions, formats)
  • Load validation rules (required fields, value ranges, cross-field checks)
  • Load prompt template for this carrier + document type combination
  • Load confidence thresholds (auto-accept, review, reject boundaries)
▼
4
Primary Extraction (Gemini)
Vertex AI (Gemini) - Config-Driven Prompts
  • Carrier-specific prompt template populated with field mappings from config
  • Gemini extracts all target fields from document
  • Per-field confidence scores generated based on extraction quality signals
  • Structured JSON output with field values, confidence scores, and source locations
▼
CONFIDENCE CHECK
Per-field confidence vs carrier thresholds
HIGH CONFIDENCE (≥ 0.9)
Auto-accepted. Proceed to validation.
MEDIUM (0.7 - 0.9)
Try Document AI fallback extraction. Re-score. If still medium: queue for human review.
LOW CONFIDENCE (< 0.7)
Route to human review queue immediately. Flag specific low-confidence fields.
▼ all paths converge ▼
5
Rule-Based Validation
Cloud Run - Config-Driven Rules
  • Required field presence check (carrier-specific required fields from config)
  • Format validation (dates, currencies, account numbers, phone numbers)
  • Cross-field consistency checks (e.g., line items sum to total, dates are chronological)
  • Business rule validation (e.g., rate within expected range for carrier/service type)
  • Validation failures: auto-flag for review with specific failure reason
▼
6
Validated Output Delivery
BigQuery + Cloud Run API + Pub/Sub
  • Validated extraction results written to BigQuery (carrier-normalized schema)
  • Structured output available via API (JSON) and export (CSV)
  • Processing complete notification via Pub/Sub to downstream systems
  • Full audit trail: document ID, extraction method, model version, confidence scores, validation results, reviewer (if any)
▼
7
Evaluation & Feedback Loop
Vertex AI Pipelines + BigQuery
  • Daily automated comparison: extraction outputs vs ground truth datasets
  • Per-carrier, per-document-type, per-field accuracy metrics computed and stored
  • Human review corrections automatically tagged and aggregated for model improvement
  • Drift detection: confidence distribution shifts trigger investigation and auto-adjustment
  • ↑ Corrections feed back to Step 4 (prompt refinement) and Step 3 (config updates)
