Evaluation Pipeline & Self-Healing Architecture
Continuous quality assurance with automated drift detection, fallback strategies, and feedback-driven improvement
A Continuous Evaluation Pipeline
Ground Truth Management
Cloud Storage + BigQuery
  • Human-verified extraction outputs per carrier per document type
  • Minimum 50 ground truth docs per carrier
  • Versioned datasets - new GT versions per quarter
  • Tagged by difficulty: standard, edge-case, adversarial
50+ verified docs per carrier for baseline
Automated Accuracy Measurement
Vertex AI Pipelines
  • Daily batch: run extraction on GT docs, compare field-by-field
  • Metrics: precision, recall, F1 per field per carrier
  • Character-level accuracy for text fields
  • Exact match for structured fields (dates, amounts)
  • Results stored in BigQuery with full lineage
>95% field-level accuracy target
Regression Detection
BigQuery + Cloud Run
  • Pre-promotion: new model version must pass accuracy gate vs current production baseline
  • No carrier can regress >1% on any field
  • Overall accuracy must meet or exceed current production
  • Automated blocking: promotion halted if regression detected
<1% max allowed regression per field
Eval Dashboards
Looker Studio
  • Accuracy heatmap: carriers x fields
  • Trend lines: accuracy over time per carrier
  • Model version comparison view
  • Human review volume and correction patterns
  • Drill-down to individual document failures
B Self-Healing Mechanisms

Cloud Run

0.5 - 0.7Route to human review queueREVIEW
< 0.5Flag as extraction failureFAIL
Drift Detection & Alerting
BigQuery ML + Cloud Monitoring
  • Statistical monitoring: track confidence score distributions per carrier using rolling 7-day windows
  • Anomaly detection: alert when mean confidence drops >5% or std deviation increases >20%
  • Field-level tracking: detect which specific fields are degrading
  • Root cause signals: correlate drift with document format changes, new carrier templates, or model updates
  • Alert escalation: Slack/email notification to ops team with specific carrier, field, and timeframe
Auto-Retry & Strategy Switching
Cloud Run + Pub/Sub
  • Retry with alternate prompt: if primary prompt fails, try backup prompt template
  • Retry with alternate model: Gemini Pro → Gemini Flash → Document AI
  • Temperature adjustment: retry with lower temperature for more deterministic output
  • Chunk strategy: if full-page extraction fails, try section-by-section extraction
  • Max 3 retries before routing to human review
Circuit Breaker
Cloud Run + Cloud Monitoring
  • Trigger: if a carrier's extraction accuracy drops below 80% over 24h window
  • Action: pause automated processing for that carrier
  • Notification: immediate alert to ops team with full diagnostic report
  • Recovery: manual investigation, config update, re-run eval pipeline to confirm fix before resuming
  • Prevents: bad data from propagating to downstream systems during quality incidents

Human Review Feedback
Cloud SQL + Cloud Run
  • Every human correction is captured with field-level diff
  • Corrections tagged by type: wrong value, missing field, wrong format, misclassified
  • Weekly aggregation: identify systematic patterns across corrections
  • Top correction patterns surface prompt improvement opportunities
Prompt Refinement Pipeline
Vertex AI Pipelines
  • Aggregate correction patterns into candidate prompt modifications
  • A/B test: route 10% of traffic through candidate prompt
  • Compare candidate vs production accuracy on same documents
  • Auto-promote if candidate accuracy ≥ production + 1% with no regressions
  • Prompt version history with full rollback capability
Active Learning
BigQuery + Vertex AI
  • Identify documents where model is most uncertain (confidence near decision boundary)
  • Prioritize these for human review to maximize learning per review
  • Expand ground truth with reviewed edge cases
  • Focus improvement effort where it matters most
New Carrier Validation Gate
Vertex AI Pipelines
  • Before any new carrier goes live, must pass eval gate:
  • ≥ 95% accuracy on 50+ test documents
  • All required fields extracted with ≥ 0.9 confidence
  • Validation rules passing at ≥ 98%
  • Automated report generated for go/no-go decision
D MLOps & Model Lifecycle Management
Model Versioning
Vertex AI Model Registry
  • Every model version tagged with: accuracy metrics, training data version, prompt version
  • Rollback to previous version within 15 minutes
  • Immutable audit log of all model deployments
A/B Testing Framework
Cloud Run + BigQuery
  • Route configurable % of traffic through candidate model/prompt
  • Real-time accuracy comparison dashboard
  • Statistical significance testing before promotion
  • Automatic rollback if candidate underperforms
Retraining Triggers
Vertex AI Pipelines
  • Trigger 1: accuracy drift exceeds threshold for 3 consecutive days
  • Trigger 2: new ground truth batch added (quarterly minimum)
  • Trigger 3: new carrier onboarded (expand training coverage)
  • Trigger 4: manual trigger by ops team for targeted improvement
Prompt Management
Cloud SQL + Cloud Run
  • Version-controlled prompt templates per carrier per doc type
  • Track which prompt version produces which accuracy per carrier
  • Prompt changelog with rationale for each modification
  • Carrier-specific prompt inheritance: base template + carrier overrides
